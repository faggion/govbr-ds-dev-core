Visão Geral

Para garantir que o seu projeto seja bem-sucedido, é importante seguir as boas práticas e recomendações disponíveis nesta seção. Os guias abrangem diversos aspectos do design system, como boas práticas em HTML, CSS e JavaScript, uso do WAI-ARIA, construção de componentes e acessibilidade. Seguindo essas recomendações, é possível aumentar as chances de sucesso no seu projeto e oferecer uma experiência positiva aos seus usuários.


Manuais Orientadores

Tenha acesso aos manuais de diretrizes, de migração e publicação, entre outros.

Boas Práticas de CSS

Aqui são abordados temas como pré-processadores de CSS, reset CSS, arquitetura css, media queries, e outros tópicos.

Boas Práticas de HTML

Esse documento propõe um padrão de codificação para a linguagem HTML5.

Acessibilidade no HTML

Esse documento tem o objetivo de definir boas práticas de construção do código HTML seguindo as diretrizes do Modelo de Acessibilidade em Governo Eletrônico - eMAG.

Codificação JavaScript

Esse documento descreve o padrão de codificação para a linguagem JavaScript utilizado no Design System de Governo.

Codificação SASS

Acesse neste tópico as orientações sobre codificação SASS, como: padronização, function e mixins, componentes e mais.

Uso do WAI-ARIA

Esse documento apresenta uma breve introdução ao Accessible Rich Internet Application - ARIA, desenvolvido pela Web Accessibility Initiative - WAI.

Navegadores Suportados

Aqui você encontra orientações sobre as versões dos navagadores suportados para o projeto design system.

Construção de Componentes

Tenha acesso às orientações sobre construção de componentes, como: regras, estruturas de pastas, PUG, JS, SCSS, e outros assuntos.

Migração da Versão 2x para 3x

Este guia vai te ajudar a migrar da versão 2.4.3 para a 3.0.0 do Design System, caso tenha alguma dúvida, poderá ser submetida através do “Formulário para registro de dúvidas” disponível na página Como Contribuir.
